  • The Internet

    Definitions:
    • HTTP = communications protocol used to connect to web servers
      Client = a computer that uses another program to transmit requests, or a piece of hardware or software that uses a server to access a service
      Server = provides a service or resource to clients in a network
      Web Browser = it is an application software to access the world wibe web or local website
      Website = A website is a collection of web pages and related content that is published on at least one web server given a common domain name.
      Web page = A web page is a hypertext document on the World Wide Web. A web server sends web pages to the user, who then sees them on a web browser.
      JavaScript = is a frontend programming lanuague used along html
      HTML = The common markup language used to create documents that can be viewed in a web browser
      CSS = is used to style a Web page
      URL = is the web address
      IDE / text editor = tool used to compile and debug code
      Command line / Terminal = is interface in which a user can enter commands for the computer to process
      BASH = is Unix shell command line integration tool
      Operating system = is a type of system software that controls computer hardware, software resources, and offers standard services to applications.
      Algorithm = is a series of instructions that must be followed in a certain order in order to generate the intended outcome.
      Frontend vs Backend Development = Front-end web development is the process of creating a website's graphical user interface using HTML, CSS, and JavaScript so that users can view and interact with it. Back-end developers ensure the website performs correctly, focusing on databases, back-end logic, application programming interface (APIs), architecture, and servers.
      Programming vs Coding = Writing codes that a machine can comprehend is the focus of the programming activity known as coding. The process of programming involves approving codes to develop programs.
      Git vs Github = Git is a version control system. GitHub is a web based repository system.
      Github terms: push, pull, fork, clone, repository =
    • push = is used to upload local repository content to a remote repository
    • pull = is used to fetch and download content from a remote repository and immediately update the local repository to match that content
    • fork = is copy of an existing repository in which the new owner disconnects the codebase from previous committers
    • clone = allows to copy an existing remote repository onto local drive
    • repository = contains a collection of files of various different versions of a project
